# 反柯里化（Uncurrying） 是柯里化（Currying）的逆过程。
核心概念
柯里化：将一个接受多个参数的函数 f(a, b, c) 转化为一系列只接受一个参数的函数 f(a)(b)(c)。
反柯里化：将一个原本设计为只接受一个参数（通常是对象本身）的方法，转化为可以接受多个参数（第一个参数作为上下文 this，后续作为实际参数）的普通函数。
简单来说，反柯里化就是“把方法从对象中提取出来，让它能独立运行，并显式地接收 this 作为第一个参数”。

# 为什么要反柯里化？
在 JavaScript 中，很多原生方法是挂在原型上的，必须通过对象调用（依赖 this）。
例如：'hello'.toUpperCase() 或 [1, 2].push(3)。
如果你想把这些方法当作普通函数传递（比如传给 map 或作为回调），直接写 Array.prototype.push 是会丢失 this 上下文的。反柯里化就是为了解决这个问题，让方法脱离对象独立存在。

# 场景对比
原始方式（依赖对象调用）：
```js const str = 'hello';
console.log(str.toUpperCase()); // "HELLO"
// 必须依附于 str 对象
```

反柯里化后（作为独立函数）：
```js
// 假设 uncurry 是反柯里化函数
const toUpperCase = uncurry(String.prototype.toUpperCase);

console.log(toUpperCase('hello')); // "HELLO"
// 不需要对象，'hello' 变成了第一个参数（即原来的 this）
```
# 如何实现反柯里化？
反柯里化的核心是利用 Function.prototype.call 或 apply，将传入的第一个参数作为 this 绑定到原方法上。
```js
通用实现代码
/**
 * 反柯里化函数
 * @param {Function} fn - 需要反柯里化的原型方法
 * @returns {Function} - 新函数，第一个参数作为 this，其余作为 args
 */
function uncurry(fn) {
  return function(...args) {
    // args[0] 将成为原方法的 this
    // args.slice(1) 将成为原方法的参数
    return Function.prototype.call.apply(fn, args);
  };
}
```
代码解析：
  fn 是类似 Array.prototype.push 这样的方法。
  返回的新函数接收任意参数 ...args。
  Function.prototype.call.apply(fn, args) 等价于 fn.call(args[0], args[1], args[2]...)。
  这里利用 call 的特性：第一个参数绑定 this，后面的是实参。
  因为 call 本身也需要动态参数，所以再用一次 apply 把 args 数组展开传进去。


# 实战示例
1. 数组方法反柯里化
```js
原生写法需要数组对象：
const arr = [1, 2];
arr.push(3); 
反柯里化后，可以把 push 当作普通函数：
const uncurriedPush = uncurry(Array.prototype.push);

const arr = [1, 2];
uncurriedPush(arr, 3, 4); 

console.log(arr); // [1, 2, 3, 4]
// 此时 arr 是作为第一个参数传进去的，充当了 this
```

2. 字符串方法反柯里化
```js
const uncurriedSlice = uncurry(String.prototype.slice);
const uncurriedToUpperCase = uncurry(String.prototype.toUpperCase);

const str = "hello world";

// 像操作普通数据一样操作字符串方法
console.log(uncurriedToUpperCase(str));           // "HELLO WORLD"
console.log(uncurriedSlice(str, 0, 5));           // "hello"
```
3. 在函数式编程中的应用
```js
当你想要组合函数时，反柯里化非常有用。例如，想检查一个数组是否包含某个值：
// 没有反柯里化，很难直接用在 map 或 filter 中作为独立逻辑
const hasValue = uncurry(Array.prototype.includes);

const list = [[1, 2], [3, 4], [5, 6]];
// 检查每个子数组是否包含数字 2
const result = list.map(subArr => hasValue(subArr, 2)); 

console.log(result); // [true, false, false]
```
# 总结：柯里化 vs 反柯里化
| 特性 | 柯里化 (Currying) | 反柯里化 (Uncurrying) |
| :--- | :--- | :--- |
| 方向 | `f(a, b)` → `f(a)(b)` | `obj.method()` → `method(obj, ...)` |
| 目的 | 固定部分参数，生成专用函数；函数组合。 | 提取原型方法，使其脱离对象独立运行。 |
| This 处理 | 通常不涉及 `this` 绑定问题，侧重参数拆分。 | 核心就是解决 `this` 绑定，将 `this` 显式化为第一个参数。 |
| 典型场景 | 配置化函数、React 高阶组件、数学计算。 | 函数式编程工具库（如 Lodash）、通用工具函数提取。 |


一句话概括：
柯里化是让函数“更挑食”（一次只吃一个参数），反柯里化是让对象的方法“不挑食”（谁调用都行，只要把对象作为第一个参数喂给它）。
