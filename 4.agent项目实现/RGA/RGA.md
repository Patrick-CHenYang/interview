# RGA是什么？

你可以把 RAG（检索增强生成）想象成是一个“开卷考试”的过程：

1. 准备阶段：把教科书（文档）拆分成小段，给每一段写上“索引号”（向量化），存进图书馆（向量数据库）。
2. 考试阶段：
  老师出题（用户提问）。
   你把题目也转化成“索引号”（提问向量化）。
   拿着这个号去图书馆找最相似的几页书（向量匹配/检索）。
   把这几页书的内容 + 题目，一起交给学霸（LLM 大模型）。
   学霸读完书后，写出最终答案。

rga功能实际上就是将用户的提问转换为向量，然后和提前转变为向量的文档去做匹配，匹配到相应的向量之后，通过向量找到对应的“原始文本”，然后把原始文本喂给大模型。
因为向量本身是一串数字（比如 [0.12, -0.45, ...]），人类和大模型都读不懂这串数字的含义，它只是用来计算相似度的。真正的知识存在于原始文本中。

## 向量匹配到后

- 大模型怎么知道该读哪篇文档？（是不是大模型自己去查的？）
- 核心揭秘：大模型并不负责“找”文档，它只负责“读”文档。
- 先遣部队：在数据发给大模型之前，嵌入模型 (Embedding Model) 和 向量数据库 已经完成了工作。
- 流程：
  1. 用户提问 -> 转为问题向量。
  2. 向量数据库 -> 通过数学计算（相似度搜索）找到最匹配的文档片段向量。
  3. 系统代码 -> 取出这些片段对应的原始文本。
  4. 组装 -> 将“原始文本” + “用户问题”打包成一个新的 Prompt。
  5. 发送 -> 此时才调用大模型，让它基于给定的材料回答。

### 结论：决定“喂什么”的是检索算法，大模型只是一个被动的、高智商的“阅读器”。

## 嵌入模型 Embedding Model

- 哥哥（嵌入模型 Embedding Model）：是个“分类专家”。它只负责把文字变成数字（向量），不会说话，不会写文章。
- 弟弟（大语言模型 LLM）：是个“创作专家”。它负责理解上下文、逻辑推理、写文章、回答问题。
虽然它们底层的技术架构（通常都是 Transformer 架构）很像，甚至有时候是用同一套基座训练出来的，但在 RAG 系统中，它们的任务和输出有着本质的区别。

  | 特性       | 嵌入模型 (Embedding Model)                               | 大语言模型 (LLM)                       |
  | -------- | ---------------------------------------------------- | --------------------------------- |
  | 主要任务     | 翻译：把文字变成一串数字（向量）。                                    | 生成：根据输入的文字，预测并生成新的文字。             |
  | 输出结果     | 固定长度的向量 例如：`[0.12, -0.45, 0.89, ..., 0.03]` (768个数字) | 自然语言文本 例如：“公司年假政策是每年15天...”       |
  | 能否对话     | ❌ 不能。你问它“你好吗”，它只会吐出一串数字。                             | ✅ 能。它可以和你聊天、写代码、写诗。               |
  | 在RAG中的作用 | 索引员：负责把问题和文档变成数学坐标，以便进行相似度匹配。                        | 回答者：负责阅读检索到的文档，组织语言回答用户。          |
  | 计算成本     | 🟢 低。速度快，便宜，适合大规模处理。                                 | 🔴 高。速度慢，贵，适合处理复杂的推理任务。           |
  | 典型代表     | `text-embedding-3-small`, `bge-m3`                   | `GPT-4`, `Claude 3.5`, `Qwen-2.5` |


- **嵌入模型是大模型吗？**
  - 从**技术出身**看：是的，它们往往同宗同源。
  - 从**功能定义**看：**不是**。大家口中常说的“大模型”（如 ChatGPT）特指能**生成文本**的 LLM。而嵌入模型只是一个**工具模型**，专门用来做数学转换。
- **在 RAG 中能不能只用一个大模型干所有活？**
  - **理论上可以，但效果极差且太贵**。如果你让 GPT-4 去把几百万份文档都读一遍并记住它们的向量位置，那成本会天价，而且速度会慢到无法接受。
  - **最佳实践**：用**便宜、快速**的嵌入模型做检索（海选），用**昂贵、聪明**的大模型做生成（决赛）。这就是 RAG 高效的原因。
